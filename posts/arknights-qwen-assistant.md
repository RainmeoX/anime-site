# 用 Qwen 给明日方舟干员做个 AI 助手

> 从干员资料数据集到 RAG 问答系统，记录我用 Qwen 模型给明日方舟做干员助手的全过程。

## 为什么做这个

明日方舟是我玩了很久的游戏，干员数量已经超过 300 位。每次想查某个干员的技能、档案、立绘，都要翻 wiki 或者游戏内图鉴，很不方便。

于是我想：**能不能用 AI 做一个干员问答助手？** 问它"棘刺的精二材料是什么"、"银灰的三技能伤害计算"，它直接回答。

## 技术选型

| 组件 | 选择 | 理由 |
|---|---|---|
| 大模型 | Qwen2.5-7B-Instruct | 中文好，开源，本地能跑 |
| 向量检索 | ChromaDB | 轻量，Python 友好 |
| Embedding | bge-small-zh | 中文检索效果好 |
| 后端 | FastAPI | 异步，文档自动生成 |
| 前端 | Vue 3 | 响应式，组件化 |

## 数据准备

数据是 RAG 的核心。我从多个来源整理了干员资料：

```python
# 数据结构示例
{
    "name": "棘刺",
    "rarity": 6,
    "profession": "近卫",
    "subprofession": "术战者",
    "skills": [
        {"name": "攻击力强化·γ型", "description": "..."},
        {"name": "护盾阻击", "description": "..."},
        {"name": "虚影", "description": "..."}
    ],
    "archive": ["档案资料一", "档案资料二", "..."],
    "tags": ["近战", "物理", "生存"]
}
```

整理成 Markdown 分块，每块约 500 字，方便向量检索。

## RAG 流程

```
用户提问 → bge 向量化 → ChromaDB 检索 Top5 → 拼接 Prompt → Qwen 生成回答
```

关键 Prompt：

```
你是明日方舟干员助手。根据以下资料回答问题，如果资料中没有答案，请说"暂无相关信息"。

资料：
{retrieved_docs}

问题：{question}
```

## 效果

问：**棘刺的精二材料是什么？**
答：棘刺精二需要：龙门币×18万、近卫双芯片×4、聚合剂×5、白马醇×7、酮凝集组×10...

问：**银灰和棘刺谁更适合打高甲？**
答：银灰的三技能"真银斩"是物理群攻，对高甲单位效果一般；棘刺的"虚影"是单体术战，对高甲更有效...

## 部署

后端用 FastAPI 跑在服务器上，前端 Vue 部署到 Vercel。模型用 vLLM 加速推理，单卡 4090 能跑到 30 tok/s。

## 总结

这个项目让我学到：
1. **数据质量比模型大小重要** —— 干净的档案资料比更大的模型更有效
2. **Prompt 工程很关键** —— 加上"暂无相关信息"约束，能大幅减少幻觉
3. **小模型 + 好数据 = 好效果** —— 7B 模型在这个任务上够用了
